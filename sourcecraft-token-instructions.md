# Регламент выдачи токена разработчика приглашенному аккаунту SourceCraft

## Для репозитория: `cloudapp-gatsby`

### Полный регламент на стороне владельца репозитория:

1. **Подготовка к выдаче токена:**
   - Убедиться, что приглашенный пользователь зарегистрирован в SourceCraft
   - Проверить, что у приглашенного пользователя есть верифицированный email
   - Подготовить список необходимых прав доступа для приглашенного разработчика

2. **Создание приглашения:**
   - Перейти в настройки репозитория в SourceCraft
   - Открыть раздел "Участники" или "Доступ"
   - Нажать "Пригласить участника"
   - Ввести email или имя пользователя приглашенного в SourceCraft
   - Выбрать уровень доступа (разработчик/участник/администратор)

3. **Настройка прав доступа:**
   - Определить уровень доступа к репозиторию:
     * **Чтение** - просмотр кода, клонирование
     * **Запись** - создание веток, пуш изменений
     * **Администрирование** - управление настройками, ветками, участниками
   - Настроить доступ к CI/CD пайплайнам при необходимости
   - Определить доступ к секретам и переменным окружения

4. **Отправка приглашения:**
   - Сформировать персонализированное приглашение с описанием проекта
   - Указать срок действия приглашения (если применимо)
   - Отправить приглашение через интерфейс SourceCraft или email

5. **Контроль и мониторинг:**
   - Отслеживать принятие приглашения
   - Проверить успешную настройку доступа
   - Обеспечить сопровождение при возникновении проблем

### Регламент действий на стороне приглашенного разработчика:

1. **Получение приглашения:**
   - Проверить email на наличие приглашения от SourceCraft
   - Перейти по ссылке приглашения или войти в SourceCraft для его принятия
   - Принять приглашение и подтвердить участие в проекте

2. **Настройка локального окружения:**
   - Установить Git и настроить его с личными данными
   - Установить необходимые инструменты разработки (Node.js, npm и т.д.)
   - Клонировать репозиторий с использованием предоставленных прав доступа

3. **Настройка аутентификации:**
   - Создать персональный токен доступа в настройках профиля SourceCraft
   - Сохранить токен в безопасном месте (менеджер паролей)
   - Настроить Git credential helper для хранения токена

4. **Проверка доступа:**
   - Выполнить тестовый клон репозитория
   - Проверить возможность создания веток
   - Проверить возможность пуша изменений в тестовую ветку
   - Убедиться в доступе к CI/CD пайплайнам (если требуется)

5. **Ознакомление с проектом:**
   - Изучить README.md и документацию проекта
   - Понять структуру репозитория и используемые технологии
   - Ознакомиться с правилами ведения кода и коммитов
   - Проверить настройки линтеров и форматирования

### Инструкции по созданию токена доступа:

1. **Создание токена в SourceCraft:**
   - Войти в свой аккаунт SourceCraft
   - Перейти в настройки профиля (Profile Settings)
   - Открыть раздел "Токены доступа" (Access Tokens)
   - Нажать "Создать новый токен" (Create New Token)
   - Указать имя токена (например, "cloudapp-gatsby-dev")
   - Выбрать уровень доступа (рекомендуется минимально необходимый)
   - Установить срок действия (если требуется)
   - Скопировать токен и сохранить в безопасном месте

2. **Настройка Git для использования токена:**
   - Открыть командную строку
   - Перейти в каталог репозитория
   - Выполнить команду: `git config --local credential.helper store`
   - При первом пуше или пуше система запросит логин и пароль
   - Ввести имя пользователя SourceCraft и токен в качестве пароля
   - Учетные данные будут сохранены в зашифрованном виде

### Задача для приглашенного разработчика:

**Цель:** Настроить доступ к репозиторию cloudapp-gatsby в SourceCraft и проверить возможность разработки.

**Задачи:**
1. Принять приглашение в проект cloudapp-gatsby
2. Создать персональный токен доступа в SourceCraft
3. Клонировать репозиторий и настроить Git для работы с ним
4. Проверить возможность создания веток и пуша изменений
5. Ознакомиться с документацией проекта и CI/CD настройками
6. Сообщить о готовности к работе

**Критерии выполнения:**
- Репозиторий успешно клонирован
- Установлены права на запись в репозиторий
- Настроен Git credential helper для хранения токена
- Выполнена проверка возможности пуша в тестовую ветку
- Ознакомление с документацией проекта завершено

**Срок выполнения:** 24 часа с момента принятия приглашения

**Поддержка:** При возникновении проблем обращаться к владельцу репозитория через систему задач SourceCraft.
